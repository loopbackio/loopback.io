Class Naming Convention

All classes use a hyphenated naming convention to separate words (i.e. .c-github-follow).

Prefix Purpose
.c- Classes that start with .c- are components. The highest level of a module, it should describe an independent module that you are creating. Components should be able to exist on their own or within other components.
._ Classes that start with ._ are subcomponents. These are a child elements of a component.
.- Classes that start with .- are component modifiers. These are used to modify components or subcomponents. They should always be compounded with a component or subcomponent, never declared as stand-alone style.
.x- Classes that start with .x- are generic class names that are neither a component or subcomponent. Most commonly these classes are used to identify defined states (i.e. .x-hide) or a generic entity that is not a component (i.e. .x-base-h1).
.js- These classes are to be avoided in our stylesheets. No exceptions!

Headings

.c-heading - Apply to any level of heading to control style independently from hierarchy.

.c-heading.-h2 - Example modifier for level 2 heading style.
